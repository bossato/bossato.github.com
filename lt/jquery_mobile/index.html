jQuery Mobileの紹介

PC、スマートフォンに対応したユーザインターフェースフレームワーク

  1. 特徴
  2. HTMLが分かれば使える
  3. タッチデバイス向けのUIが簡単に作れる
  4. 広範囲なプラットフォームとの互換性
    • Apple iOS 3.2-5.0
    • Android 2.1-2.3、3.1、4.0
    • Windows Phone 7-7.5
    • Chrome Desktop 11-15、Firefox Desktop 4-8 、Internet Explorer 7-9
      • 参考URL:http://dev.screw-axis.com/doc/jquery_mobile/overview/supported_platforms/

最新バージョン：1.1.0

jQuery Mobileの使い方

準備

以下のファイルを読み込む 
<head>
<!-- jQuery MobileのCSSを読み込む -->
<link rel="stylesheet" href="jquery.mobile.css" />
<!-- jQuery本体を読み込む -->
<script src="jquery.js"></script>
<!-- jQuery MobileのJSを読み込む -->
<script src="jquery.mobile.js"></script>
</head>

ページ構造

ページ要素の中にヘッダー、コンテンツ、フッター要素を入れる
複数ページある場合はページ要素タグにidをつける 
<div data-role="page" id="p1"> 
	<div data-role="header">...</div> 
	<div data-role="content">
		<a href="#p2">ページ２へ</a>
	</div> 
	<div data-role="footer">...</div> 
</div>

Tipsや注意点など

UIについて

jQueryMobileはあくまでUIフレームワーク

  1. リッチな表示(カルーセルとか)が出来ると勘違いされやすい
  2. JSプラグインで追加することも出来る

jQueryMobileのUI(CSS)をいじるのはそれなりに大変

  1. あるパーツを変更したいだけなのに他のパーツまで変更されてしまうことがある(ヘッダーのCSS変更するとリストヘッダーまで変わるなど)
  2. パーツごとにID付ける、それようのテーマを作るなどの対応が必要になってくる
  3. 早めの段階でこういうUIまでがjQueryMobileで出来る、それ以上のことをやる場合は追加になるという調整をした方が後々幸せになれるよきっと

Androidについて

挙動がちょっと不安定かもしれない。。

  1. Ajax処理、アニメーション処理が端末によって挙動が不安定(感覚的には端末依存問題な気がする)
  2. Android OSの幅広いバージョンがサポートされていても端末依存の問題はさけられない(jQuery Mobileに限らず)
  3. 早めの段階でAndroidの挙動に対して調整をした方が後々幸せに(ry

Ajaxについて

Ajaxによって処理が重たいと言われてる

  1. Ajax処理自体を無効にする流れが出来てます
    2. <script language="JavaScript">
	$.mobile.ajaxEnabled = false;
</script>
  2. そんなあなたに読んでいただきたい記事が
  3. ページ表示が重いという時は以下も合わせて確認した方がよいと思います
    4. CSS、JSの順に外部ファイルを読み込む
CSSで@importを使用しない
アイコンやボタン等の画像でCSSスプライトを用いる
CSS、JSは圧縮する

Ajax無効はせっかくのjQueryMobileの機能を削ることになるので逃げちゃダメ
